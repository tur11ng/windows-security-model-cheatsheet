Part I

Foundations — The Windows Security Model

How the kernel enforces identity and access. Pure engineering reference: the components that exist, the data structures they manipulate, and the rules they follow when an access decision is made.

01 — Windows Security Architecture

ARCH
Windows security model Access control components
Kernel Security Components
  • SRMSecurity Reference Monitor — sole kernel authority for enforcing access control. All object open requests pass through SRM's access check algorithm.
  • Object ManagerManages all kernel objects (files, processes, mutexes, tokens…) and their security descriptors. Controls handle creation and duplication.
  • ExecutiveCollection of kernel subsystems: Process Manager, Memory Manager, I/O Manager, Configuration Manager (registry), ALPC.
  • TokenEvery process/thread carries an access token — the SRM's identity carrier.
User-Mode Security Processes
  • LSASS.exeLocal Security Authority Subsystem. Hosts authentication packages, manages logon sessions, enforces LSA policy, stores credential secrets in protected memory.
  • Winlogon.exeOrchestrates interactive logon. Captures Secure Attention Sequence (Ctrl+Alt+Del). Creates user desktops. Calls LsaLogonUser.
  • SCM (services.exe)Service Control Manager. Launches services with configured tokens (SYSTEM, Local Service, Network Service, or named accounts).
  • SAM (samss)Security Account Manager — local user/group database. Hosted in LSASS. Backed by HKLM\SAM.
Authentication Packages (in LSASS)
  • MSV1_0NTLM authentication. Local accounts, NTLMv1/v2.
  • KerberosKerberos v5. Domain accounts. Default for domain-joined.
  • NegotiateSPNEGO broker. Selects Kerberos first, falls back to NTLM.
  • TSSSP / CredSSPRDP NLA — delegates full credentials to remote host over TLS.
  • WDigestDeprecated. Originally stored cleartext passwords in LSASS memory.
Key Security Databases
  • HKLM\SAMLocal accounts, NT hashes, group memberships. ACL: SYSTEM only.
  • HKLM\SECURITYLSA secrets (service passwords, cached domain credentials, DPAPI master keys). ACL: SYSTEM only.
  • NTDS.ditAD domain database on DCs. Contains all domain user hashes, objects, attributes.
  • SYSVOLGroup Policy templates. Replicated to all DCs via DFSR/FRS.
Security Boundaries
  • ForestTrue security boundary in AD. Separate forests cannot implicitly trust each other.
  • Kernel/User modeRing 0 vs Ring 3. User mode cannot directly bypass SRM or write to kernel memory.
  • Integrity LevelMandatory control. Lower IL processes cannot write to higher IL objects.
  • AppContainerSandboxed IL=Low + capability-gated access. UWP apps, browsers.
