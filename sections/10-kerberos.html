10 — Kerberos Authentication

KRB5
Kerberos overview

Kerberos v5 is the default protocol for domain authentication. It is ticket-based and mutual — both client and server authenticate each other. The KDC (Key Distribution Center) runs on every Domain Controller. No passwords travel the network; all encryption uses keys derived from account passwords.

Components
  • KDCKey Distribution Center. Runs on DC. Contains AS + TGS + database of all account keys.
  • ASAuthentication Service. Issues TGTs in response to valid pre-auth.
  • TGSTicket Granting Service. Issues service tickets in exchange for valid TGTs.
  • PrincipalAny identity: user, computer, or service account.
  • SPNService Principal Name. Format: service/host@REALM.
  • RealmDomain (uppercase). Cross-realm trusts enable inter-domain auth.
  • krbtgtSpecial account whose hash encrypts ALL TGTs.
Ticket Types & Lifetime
  • TGTTicket Granting Ticket. Issued by AS. Encrypted with krbtgt hash. Default 10 hours, renewable up to 7 days.
  • Service TicketIssued by TGS. Encrypted with target service account's hash.
  • PACPrivileged Attribute Certificate embedded in tickets. Contains user SID, group SIDs, logon info. Signed by KDC.
  • AuthenticatorClient-generated timestamp encrypted with session key. Sent with each ticket use to prevent replay.
Encryption Types
  • RC4-HMACUses NT hash directly as key. Legacy.
  • AES-128/256Uses PBKDF2-derived key from password + salt (username@domain). Modern standard.
  • DES-CBC-CRCLegacy, disabled by default since Win7/2008R2.

Pre-authentication

By default, the AS-REQ includes a timestamp encrypted with the user's long-term key. This proves the client knows the password before the KDC issues a TGT. Without it, the AS-REP becomes offline-crackable.

Kerberos Delegation Models

Delegation allows a service to act on behalf of a user against another service — necessary for multi-tier applications.

  • Unconstrained User's TGT is forwarded to the front-end service and cached there. The service can request tickets to any downstream service. Configured by TrustedForDelegation on the account object.
  • Constrained (KCD) Front-end service can only delegate to a specific list of SPNs in msDS-AllowedToDelegateTo. Uses S4U2Self (obtain a self-ticket for any user — protocol transition) and S4U2Proxy (use that ticket to get one for the back-end service).
  • Resource-Based (RBCD) Configured on the back-end resource via msDS-AllowedToActOnBehalfOfOtherIdentity. The resource lists which front-end principals may delegate to it.
  • PKINIT Certificate-based Kerberos pre-authentication. Smart cards and ADCS-issued certificates. The KDC trusts certificates issued by an Enterprise CA mapped to a user account.
Sensitive Account Protection
Adding accounts to the Protected Users group disables NTLM auth, unconstrained delegation, DES/RC4 Kerberos, and CredSSP caching. TGT lifetime is forced to 4 hours, non-renewable. The Account is sensitive and cannot be delegated flag prevents an account's tickets from being forwarded.
