Access Masks & Access Rights

AM
Access-mask format Generic access rights Standard access rights GENERIC_MAPPING

Every OpenXxx call and every ACE carries a 32-bit ACCESS_MASK. The SRM compares the requested mask against DACL ACE masks bit-for-bit. The 32 bits are partitioned into four tiers with fixed semantics assigned by the kernel.

Generic Rights  [bits 31–28]

Coarse-grained, object-agnostic rights. Resolved by RtlMapGenericMask() before the access check runs — they are never stored in an ACE or returned in a granted-access mask.

ConstantHexBit
GENERIC_READ 0x8000000031
GENERIC_WRITE 0x4000000030
GENERIC_EXECUTE 0x2000000029
GENERIC_ALL 0x1000000028
Standard Rights  [bits 23–16]

Apply uniformly to every securable object type. Always combinable with object-specific rights in a single mask.

ConstantHexMeaning
DELETE 0x00010000Delete the object
READ_CONTROL 0x00020000Read owner, group, and DACL
WRITE_DAC 0x00040000Replace the DACL
WRITE_OWNER 0x00080000Change the owner SID
SYNCHRONIZE 0x00100000Wait on the object handle (waitable objects)
Special Rights  [bits 25–24]
ConstantHexSemantics
MAXIMUM_ALLOWED 0x02000000 Instructs the SRM to grant the largest access mask the caller's token allows. The SRM resolves it and returns the actual set of bits — the caller does not specify a desired mask.
ACCESS_SYSTEM_SECURITY 0x01000000 Read or write the SACL (audit rules) on an object. Requires SeSecurityPrivilege regardless of the DACL — it is a privilege-gated right, not a DACL-gated one.
Attacker note
MAXIMUM_ALLOWED on OpenProcess is a common malware pattern — it avoids specifying exact rights and silently receives everything the DACL permits, which defeats detection rules keyed on specific requested-access bits such as PROCESS_VM_READ | PROCESS_VM_WRITE.

Why READ / WRITE / EXECUTE all equal READ_CONTROL
The combined standard masks are intentionally identical — standard rights carry no intrinsic read/write/execute meaning. That semantics comes exclusively from the object-specific bits in [15:0].
GENERIC_MAPPING — Per-Object Resolution

Each kernel object type registers a GENERIC_MAPPING structure in its type object. RtlMapGenericMask() uses it to expand generic bits into object-specific combinations before the access check. The generic bits are consumed and never appear in a stored ACE or a returned granted-access mask.

Key invariants
Generic bits are never stored in an ACE and never returned in a granted-access mask — RtlMapGenericMask() always expands them first. MAXIMUM_ALLOWED is similarly never stored. Every AccessAllowed and AccessDenied ACE uses only object-specific bits [15:0] combined with standard rights [23:16].
