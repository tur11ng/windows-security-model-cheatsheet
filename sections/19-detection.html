Layered Hardening Stack
The most resilient posture combines: Credential Guard + LSASS PPL for credential isolation; LAPS for local account uniqueness; tiered admin model with Deny logon rights enforcing separation; SMB/LDAP signing + EPA for relay resistance; Disable NTLM where compatible; gMSA for service accounts; AD CS hardening + EPA on enrollment endpoints; BloodHound ACL audits on a regular cadence; Protected Users for all Tier-0 identities.

High-Fidelity Hunt Queries (Skeleton)
  • LSASS Handle Sysmon 10 where TargetImage contains lsass.exe AND GrantedAccess in (0x1010, 0x1038, 0x143a) AND SourceImage NOT IN (AV/EDR known paths)
  • DCSync Event 4662, Properties includes DS-Replication-Get-Changes-All GUID (1131f6ad-…), SubjectUserName is NOT a DC computer account
  • Kerberoasting Burst Event 4769, TicketEncryptionType=0x17, count(distinct ServiceName) > 5 per SourceIP per minute
  • Golden Ticket Event 4769 where no corresponding 4768 for same account in last 10 hours on any DC; OR ticket lifetime > 10 hours
  • RBCD Write Event 5136, AttributeLDAPDisplayName = msDS-AllowedToActOnBehalfOfOtherIdentity, ModifierSID NOT admin group
  • Lateral PtH Event 4624, LogonType=3, AuthenticationPackageName=NTLM, SubjectUserName matches local admin naming convention, across >2 hosts
  • Service Install (Persistence) Event 7045 or 4697, ServiceType=0x10 (WIN32_OWN_PROCESS), ServiceStartType=2 (AUTO_START), within hours of 4625/4624 anomaly
