06 — Privileges

PRIV
Privileges

Privileges are special rights in a token that override normal access checks. They must be enabled in the token (some are disabled by default and must be explicitly activated) and are distinct from access rights (which are checked against object DACLs). Privileges are assigned via User Rights Assignment security policy — see Section 07.

PrivilegeEffectDefault Holders
SeDebugPrivilegeOpen any process/thread regardless of DACL. Read/write memory of any process.Administrators (disabled)
SeImpersonatePrivilegeImpersonate any user who authenticates to the local machine, including SYSTEM via named pipe or RPC.Service accounts, Admins
SeAssignPrimaryTokenPrivilegeReplace the primary token of a process — effectively run code as another user.SYSTEM, SCM
SeCreateTokenPrivilegeCreate arbitrary access tokens with any SIDs/privileges.SYSTEM only
SeTcbPrivilegeAct as part of the OS (Trusted Computing Base). Required to create tokens for arbitrary users.SYSTEM
SeBackupPrivilegeRead any file/registry key regardless of DACL.Backup Operators, Admins
SeRestorePrivilegeWrite any file/registry key regardless of DACL.Backup Operators, Admins
SeTakeOwnershipPrivilegeTake ownership of any object (grants WriteDac → can then rewrite DACL).Administrators
SeLoadDriverPrivilegeLoad/unload kernel drivers — direct code execution in Ring 0.Administrators
SeSecurityPrivilegeRead/write SACL on any object. View/clear Security event log.Administrators
SeChangeNotifyPrivilegeBypass traversal checking (directory access check skipped). Held by all users.All users (mandatory)
SeRelabelPrivilegeRaise or lower the integrity label of an object.Administrators
SeEnableDelegationPrivilegeMark user/computer accounts as trusted for Kerberos delegation in AD.Domain Admins
SeSyncAgentPrivilegeSynchronize directory data (DirSync).SYSTEM on DCs
